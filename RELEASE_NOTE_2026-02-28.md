# Family News 릴리즈 노트 (2026-02-28)

## 개요
오늘 배포분은 **사진 업로드/수정 흐름 개선**, **상세 페이지 이미지 표시 개선**, **날짜 입력 기능 확장**, **홈 최신 반영 기준 정리**에 집중했습니다.

---

## 1) 대표사진 선택 방식 개선
### 변경 전
- 여러 장 업로드 시 첫 번째 사진이 자동으로 대표사진이 됨
- 사용자가 대표사진을 직접 지정하기 어려움

### 변경 후
- 업로드/수정 화면에서 사진 미리보기마다 `대표사진` 체크 가능
- 체크한 사진이 대표사진으로 저장됨
- 대표사진 미선택 시 저장 전에 안내 메시지 표시

### 영향 화면
- 사진 업로드
- 기사 수정

---

## 2) 사진 수정 기능 강화 (삭제/교체)
### 변경 전
- 수정 화면에서 기존 추가사진을 개별 삭제하기 어려움
- 대표사진 관리 옵션이 제한적임

### 변경 후
- `대표 사진 삭제` 체크 추가
- 기존 추가사진마다 `삭제` 체크 추가
- 새로 올리는 사진 중 대표를 체크해 교체 가능
- 대표 삭제 시 남은 사진/신규 업로드 기준으로 안전하게 대표 유지

### 영향 화면
- 기사 수정

---

## 3) 드래그 업로드 UX 안정화
### 변경 전
- 체크박스 클릭 시 드롭존 클릭 이벤트가 함께 동작하여 파일 선택창이 열림
- 드래그로 선택한 뒤에도 의도치 않게 파일창이 재오픈될 수 있음

### 변경 후
- 체크박스/라벨 클릭 시 파일 선택창 미오픈
- 드래그로 파일 선택한 상태에서는 드롭존 클릭으로 파일창 재오픈 차단
- 실수로 파일 선택 취소되는 흐름 최소화

### 영향 화면
- 사진 업로드
- 기사 수정

---

## 4) 상세 페이지 이미지 표시 개선
### 변경 전
- 상세 본문 이미지가 크게 보일 수 있어 가독성 저하
- 여러 장 사진의 집중도가 떨어질 수 있음

### 변경 후
- 슬라이드에서 한 번에 1장씩 표시 유지
- 이미지 최대 크기/슬라이드 폭 조정으로 본문 가독성 개선

### 영향 화면
- 상세 페이지

---

## 5) 날짜 기능 확장 (과거 사진 대응)
### 변경 전
- 업로드 시점 외 별도 이벤트 날짜 저장 불가
- 과거 사진의 실제 행사일 표현 어려움

### 변경 후
- 업로드 시 `촬영/작성 날짜와 시간` 선택 가능
- 별도 `이벤트 날짜` 필드 추가
- 상세 메타에서 이벤트 날짜 표시(입력 시)

### 데이터 반영
- 업로드 시 선택 시간은 작성 시각 저장에 반영
- 이벤트 날짜는 게시글 필드로 저장

---

## 6) 홈 메인/주요 소식 최신 반영 규칙 통일
### 변경 전
- 이벤트 날짜/작성 시각 조정 시 홈의 최신 노출 체감이 달라질 수 있음

### 변경 후
- 홈 메인 타이틀(히어로) + 주요 소식 모두 최신 등록 순서 기준으로 반영
- 새로 등록한 사진 소식이 홈 상단에 즉시 반영되도록 일관성 확보
